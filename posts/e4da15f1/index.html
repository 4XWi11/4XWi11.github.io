Least Significant Bit Oracle Attack （LSB Oracle Attack / Parity Oracle）

该攻击使用的条件很简单，我们可以进行解密，并且会有oracle告诉我们明文的奇偶性，时间复杂度是O(logN)O(logN)O(logN)

假设已知的为enc，我们构造密文C

C=(2e⋅c) mod n=(2e⋅me) mod n=(2⋅m)e mod nC=(2^e\cdot c)\ mod\ n=(2^e\cdot m^e)\ mod\ n=(2\cdot m)^e\ mod\ n\notag C=(2e⋅c) mod n=(2e⋅me) mod n=(2⋅m)e mod n

我们将这个密文send过去，那么显然对应的明文就是M=(2⋅m) mod nM=(2\cdot m)\ mod\ nM=(2⋅m) mod n

oracle会告诉我们M的奇偶性，也就是最低位lsb是0还是1

由此我们可以进行推导，得出的结论如下

  • 如果lsb是0，说明m<n2m<\frac{n}{2}m<2n​
  • 如果lsb是1，说明m≥n2m\geq \frac{n}{2}m≥2n​；如果2m2m2m大于nnn，但注意mmm是不会超过nnn的，所以2m2m2m就不会超过2n2n2n，则2m mod n=2m−n2m\ mod\ n=2m - n2m mod n=2m−n，nnn是奇数，2m2m2m偶数，那么MMM是奇数，lsb是1

这样一次就可以将mmm限定在n2\frac{n}{2}2n​的左边后者是右边，之后就是不断地二分，知道找到真正的m。因为运用二分的思想，所以时间复杂度才会是logloglog

接下来就是继续构造

C=(2e⋅C) mod n=(2e⋅(2⋅M)e) mod n=(4⋅m)e mod nC=(2^e\cdot C)\ mod\ n=(2^e\cdot (2\cdot M)^e)\ mod\ n=(4\cdot m)^e\ mod\ n\notag C=(2e⋅C) mod n=(2e⋅(2⋅M)e) mod n=(4⋅m)e mod n

将CCC给send过去就可以在，获得oracle，就可以将CCC限定在n4\frac{n}{4}4n​的空间里
