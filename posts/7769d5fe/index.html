0x02 信息表示与处理

按字节分

  • 单字节：字节（Bytes），符
  • 双字节：词（Word），字
  • 四字节：双词（DWord，doubleword）
  • 八字节：四词（QWorld，quadword）
  • 16字节：双四词，八词（double quadword）

对应C语言的数据类型（Fr.《深入理解计算机系统》

0x03 Python

0x102 stack overflow

0x00 保护机制

0x01 ret2shellcode

特征：栈可执行保护（NX）关闭（内存中有一段rwx的

0x02 ROP

特征：1. 程序存在栈溢出，NX保护开启，其他保护关闭；2. 找得到满足的gadgets

Return Oriented Programming（返回导向编程），利用程序已有的代码片段（gadgets）来改变某些寄存器或者变量的值，绕过NX保护

  • gadgets：以ret结尾的指令序列

0x1 ret2text

特征：程序本身有提供了shellcode等功能

只需要栈溢出将返回地址覆盖为这段代码的起始地址

0x2 ret2syscall

特征：程序没有现成的shellcode，找代码片段（gadget）

通过执行gadgets改变寄存器的值然后控制程序执行系统调用

EXP：

0x3 ret2libc

基本方法：找到system函数地址，找到/bin/sh字符串地址

最为常用，执行libc中的函数，通常是返回到某个函数的plt处或者是函数的实际地址

注意：

  1. 32位程序的栈中，一个函数地址的后面紧跟的是执行程序之后的返回地址，然后才是函数的参数

  2. 64位程序，函数也存放在寄存器RDI,RSI,RDX,RCX,R8,R9等当中，第7个参数才会放在栈中

EXP：

  • 有system和/bin/sh（分开的，用gdb plt找
  • 没有system和/bin/sh，泄漏system地址

（通过被使用函数got表中存放的真实地址与libc基址相减，偏移再和system内存地址相加，从而泄漏system真实地址

0x4 ret2dl

利用_dl_runtime_resolve函数构造payload，因为该函数是对动态链接的函数进行重定位的

0x03 Canary

特征：打开canary和NX

绕过方法：

  1. 泄漏栈中的canary

    • 有合适的输出函数
    • 泄漏canary后程序不会崩溃

    参考https://4xwi11.github.io/posts/a89e21c0/#pwn-canary

  2. 爆破canary

    • 多进程

  3. 其他方法

    联合其他漏洞修改劫持__stack_chk_file函数

    覆盖TLS中存储的Canary值

0x04 PIE

特点：开启PIE的程序每次运行时地址的后12位是固定不变的

绕过方法：

  1. Partial Write
    • 程序不大，有后门函数
    • 只需爆破一个字节
  2. 直接泄漏地址

0x06 栈迁移（pivoting）

劫持栈指针指向攻击者所能控制的内存处，进行ROP等操作，即将ESP或RSP指针修改为可以控制的内存区地址，比如bss段、堆等，然后在可控内存区当中布置payload

条件：1. 程序存在溢出；2. 可以控制sp指针

特征：1. 栈溢出的溢出空间大小不足；2. 存在较大可读可写可执行的区域

注意：远离不可读不可写的内存

总结，也就是说在bss等地方构造ROP链不是通过栈溢出实现的，而跳转到bss段上才是通过栈溢出

0x103 格式化字符串

程序代码在输出的时候直接

而且value是用户可以控制的，那就存在格式化字符串漏洞：

由于printf函数族的设计缺陷，当其第一个参数可被控制时，攻击者将有机会对任意内存地址进行读写操作

可以在调试的时候输入，读取距离栈顶第n个栈中的值并以十六进制的形式打印出来

任意地址读

既然可以泄漏栈上的内容，那么可不可以泄漏其他任意地址的内容呢？

答案是肯定的，%s的本质是读取对应的参数，并作为指针解析，获取到对应地址的字符串并输出；将上述的x换成s，就可以实现任意地址读

攻击流程如下

  1. 确定相对偏移：通过%x找到我们输入的字符串是从栈中的第几个开始的
  2. 将相应位置作为参数输入

payload的形式如下，可以实现对任意地址_addr内容的输出

任意地址写

将用到一个新的格式化字符%n

  • %n：一次写入4个字节
  • %hn：一次写入2个字节
  • %hhn：一次写入1个字节

举个🌰

正如上图运行结果所示，%n会将前面内容的字节数赋值给后面对应地址指向的内存，并且可以通过%numc%n来简便写更长字节的数据。但这样显然是不合理的，如果需要更改的数值很大，则需要输入很长的数据，这对于程序而言往往是不允许的（上图的a前有122个空格）。不过我们可以利用%hhn一个字节一个字节地修改

Windows保护机制

Windows安全中心$\rightarrow$应用和浏览器控制$\rightarrow$Exploit Projection

用winchecksec查看，Fr.https://github.com/trailofbits/winchecksec/releases/tag/v3.0.2

以下两个是地址随机数，和Linux的pie类似；注意就算编译的时候关闭，如果操作系统强制开启，实际运行的时候还是会开启该保护

  • Dynamic Base
  • ASLR
  • NX/DEP 堆栈保护，与Linux下的一样，绕过可以调用VirtualProtect函数
  • SEH异常处理链，链表结构
  • CGF（Control Flow Guard），gadget失效，只能调用完整的函数
  • SafeSEH
  • GS stack cookie，类似于linux下的canary

SEH

也就是C语言__try__finally和__try__except的底层原理
